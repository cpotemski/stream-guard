# Arbeitsregeln fuer Twitch Watcher

## Festgelegt
- Fuer dieses Projekt wird das eigene Memory-Projekt `twitch-watcher` verwendet.
- Wichtige Entscheidungen und echte Fortschritte werden fortlaufend in Memory dokumentiert.
- Design-Prinzip: YAGNI strikt einhalten.
- Es wird nur umgesetzt, was die definierten MVP-Acceptance-Criteria direkt stuetzt.

## Konsequenzen
- Keine vorgezogenen Nice-to-haves.
- Keine Architektur-Aufblaehung ohne direkten MVP-Nutzen.
- Bei Unsicherheit gilt: einfachste robuste Loesung waehlen.

## Repo-Regel
- Diese Leitplanken werden zusaetzlich in einer projektspezifischen `AGENTS.md` im Repository verankert.
