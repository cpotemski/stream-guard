# Entscheidung
Der aktive Projektfokus wird von "MVP liefern" auf "fertig nutzbare v1" umgestellt.

## Verbindliche Zielrichtung v1
- 24/7-Betrieb mit robuster Selbstheilung bei Sleep, Disconnects, Reload-Lücken und Twitch-Fehlerzuständen.
- Minimale Nutzerinteraktion: wichtige Streams verwalten, automatische Live-Überwachung, automatisches Öffnen, Laufzeitstabilisierung (nicht pausiert/nicht muted), Auto-Claim, Streak-Schutz.
- Popup als produktive Betriebsoberfläche (Übersicht, Priorisierung, relevante Stats), ohne sichtbare Debug-Oberfläche für Endnutzer.

## Konsequenzen
- MVP-bezogene Scope-Formulierungen gelten nur noch historisch.
- Doku/Memory/Plan werden auf v1 konsolidiert, inklusive Markierung veralteter oder revidierter Entscheidungen.
- Design-Prinzip bleibt: einfache robuste Lösungen, aber nicht mehr durch MVP-Akzeptanzkriterien limitiert.
